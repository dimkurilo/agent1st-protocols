# Как писать запросы для DeepSeek V4

Три простых правила. Никакой науки - только то, что работает.

---

## Правило 1: Отдели «что» от «почему»

DeepSeek нужно знать две вещи: **конкретное действие** и **контекст**.
Если они смешаны в одну простыню - агент тратит силы на разбор, а не на дело.

**Плохо** (всё слитно):
```
Работаю над аналитикой, есть файлы A и B, их надо сравнить с C,
потому что мы переходим на новую версию, и вообще контекст такой-то...
```

**Хорошо** (действие отдельно, контекст отдельно):
```
Сравни два файла с третьим:
- /path/to/A.md
- /path/to/B.md
- /path/to/C.md

Контекст: переходим с v9 на v12, нужно понять, что устарело.
```

---

## Правило 2: Проверь три вопроса перед отправкой

Перед тем как отправить - убедись, что в тексте есть ответы:

| Вопрос | Пример |
|---|---|
| **Что сделать?** | «Сравни файлы», «Найди причину ошибки», «Посчитай конверсию» |
| **Где это?** | Путь к файлу, название листа Excel, текст ошибки |
| **Что считается готовностью?** | «Дай список правок», «Выведи цифру», «Скажи, что устарело» |

Если хотя бы на один вопрос нет ответа - допиши. Одно предложение.

---

## Правило 3: Ограничения - в конце

Если что-то НЕЛЬЗЯ трогать - напиши отдельной строкой перед `【思维模式要求】`.

```
Не трогать: файлы агентов (только анализ), другие листы Excel, формат CSV.
```

Ограничения экономят время: агент не спрашивает «а можно ли мне...», а сразу работает в заданных рамках.

---

## Шаблон (если хочется структуры)

Необязательно, но если задача сложная - помогает:

```
Задача: [одно предложение - что сделать]

Где: [файлы, листы, функции]
Сейчас: [что не так - если это баг/проблема]
Должно быть: [какой результат нужен]

Контекст: [пара предложений - зачем это делается]

Не трогать: [границы - если есть]

【思维模式要求】在你的思考过程（think标签内）中，请遵守以下规则：
1. 禁止使用圆括号包裹内心独白，所有分析内容直接陈述即可
2. 禁止以第一人称描写内心活动，请用分析性语言替代
3. 思考内容只包含：约束条件、陷阱识别、工具选择、执行步骤
4. 复杂任务使用结构化分析：UNDERSTAND→ANALYZE→APPROACH→PLAN→CRITIQUE
5. 路径明确时跳过思考，立即执行工具调用
```

## Инъекция: когда нужна, когда нет

| Ситуация | Инъекция `【思维模式要求】` |
|---|---|
| Обычная задача (анализ, код, Excel) | **Всегда** - в конце сообщения |
| Простой вопрос («что делает эта функция?») | Можно без неё |

Инъекция подавляет roleplay-стиль мышления DeepSeek (внутренние монологи, «размышления вслух») и направляет его в инструментальное русло: сначала ограничения, потом план, потом действие. Без неё модель может уйти в нарратив вместо дела.

---

## Осторожно: 【】 в загружаемых файлах

DeepSeek V4 обрабатывает **любой** `【】` блок в контексте с training-level приоритетом.
Если файл, загруженный в промпт, содержит `【角色沉浸要求】` или другой `【】` блок -
он переопределит thinking mode агента. Последствия: roleplay-мышление, игнор tool calls,
деградация за 1-2 хода.

Перед загрузкой файлов в промпт - убедись, что в них нет `【】` блоков.
Если нашли - удалите или замените скобки на `[]`.

## Примеры

### Простая задача - хватит трёх строк

```
Почини баг: POST /api/orders возвращает 500
Ошибка в логах: AttributeError в serializer.py:45
Не трогать: другие эндпоинты, формат ответа

【思维模式要求】...
```

### Сложная задача - шаблон помогает

```
Задача: сравни два маркетинговых агента с базовым v12, дай разбор по доработкам

Где:
- agents/marketing/market1st_v4-analytics-deepseek.md
- agents/marketing/market1st_v4.1-creative-deepseek.md
База: agent1st_v12-deepseek.md

Контекст: агенты созданы на базе v9, для маркетинга и аналитики.
Не для кода. Интересует: не перегружены ли продуктовыми данными,
что устарело относительно v12, оправдана ли специализация.

Не трогать: файлы агентов (анализ без правок)

【思维模式要求】...
```

### Совсем просто - одной строкой

```
Переименуй `tmp` в `result_buffer` в parser.py

【思维模式要求】...
```

---

## Главное правило

**Лучше недоструктурировать, чем переструктурировать.** DeepSeek с инъекцией
сам разберётся в ходе thinking. Твоя задача - дать ему «что», «где» и «как
проверить готовность». Остальное он сделает сам.

Не надо писать по шаблону. Надо - чтобы в тексте были ответы на три вопроса:
что сделать, где, что считается результатом.

---
