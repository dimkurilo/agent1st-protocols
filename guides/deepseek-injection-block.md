# DeepSeek V4 — Инжекционный блок для инженерных задач

## Зачем это нужно

DeepSeek V4 обучен на roleplay-стиле мышления — внутренние монологи, 
«я думаю», «хм, интересно», скобочные отступления. Без инжекционного блока 
thinking содержит ~40-60% «воды» — мусорных токенов, которые не несут 
аналитической ценности.

Инжекционный блок переключает модель в **Pure Analysis** режим — чистая 
логика, без ролевой игры. Это training-native механизм: модель обучали 
реагировать именно на эти формулировки в конце первого сообщения.

**Результат**: thinking-блок в 3-4 раза короче, структурно чище, 
без «хм», «окей», «пожалуй, я сделаю вот что».

## Как использовать

**Скопируйте блок ниже и вставьте в КОНЕЦ вашего сообщения с задачей.**
Оставьте одну пустую строку между задачей и блоком.

Работает для: `deepseek-v4-pro`, `deepseek-v4-flash` (API + Expert Mode).
Не работает для: Quick Mode в веб/приложении.

## Полная версия (максимальная compliance)

```
【思维模式要求】在你的思考过程（think标签内）中，请遵守以下规则：
1. 禁止使用圆括号包裹内心独白，例如"（心想：……）"或"(内心OS：……)"，所有分析内容直接陈述即可
2. 禁止以第一人称描写内心活动，例如"我心想""我觉得""我暗自"等，请用分析性语言替代
3. 思考内容只包含：约束条件、陷阱识别、工具选择、执行步骤。禁止角色扮演式的内心戏
4. 复杂任务使用结构化分析：UNDERSTAND→ANALYZE→APPROACH→PLAN→CRITIQUE
5. 路径明确时跳过思考，立即执行工具调用
```

## Облегчённая версия (RU, compliance чуть ниже)

```
【РЕЖИМ МЫШЛЕНИЯ】В thinking-блоке строго запрещено:
1. Внутренние монологи в скобках (типа "(надо подумать...)", "(хм, интересно...)")
2. Повествование от первого лица ("я думаю", "мне кажется", "наверное стоит...")
3. Только: анализ ограничений → выбор инструмента → план действий → исполнение
4. Если путь очевиден — thinking не нужен, сразу вызов инструмента
```

## Что будет, если НЕ вставлять блок

Модель выполнит задачу в любом случае. Разница — в thinking:

| | Без блока | С блоком |
|---|---|---|
| **Стиль thinking** | Ролевой: «Хм, надо подумать...», «О, кажется тут проблема!» | Аналитический: «Constraint: signature fixed. Trap: empty valid.» |
| **Объём thinking** | ~100-200 токенов на среднюю задачу | ~30-60 токенов |
| **Вода** | ~40-60% (междометия, нарратив, скобки) | ~0-5% |
| **Структура** | Поток сознания | UNDERSTAND → PLAN → CRITIQUE |
| **Посторонние мысли** | Часто: «(может, заодно и email?)» | Нет |

**Практический эффект**: для простых задач разница незаметна. Для сложных 
(отладка, многошаговые изменения) — экономия токенов и структурная ясность 
становятся значимыми.
